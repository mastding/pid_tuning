# PID 智能整定系统帮助中心

## 1. 系统定位

本系统用于工业控制回路的智能 PID 整定，目标是把一次整定任务拆成可解释的多智能体流程，并在任务结束后自动沉淀经验，供后续类似回路复用。

系统当前支持：

- 上传 CSV 或直接获取历史数据
- 多智能体执行过程展示
- 多模型辨识与多策略整定
- 闭环仿真评分与自动回流优化
- 经验中心与相似经验检索

## 2. 前端页面

### 2.1 智能整定页

智能整定页负责发起整定任务并查看执行过程。

支持的数据来源：

- 上传 CSV
- 获取历史数据

支持的回路类型：

- 流量
- 温度
- 压力
- 液位

任务执行时，页面会显示：

- 用户任务描述
- 多智能体执行过程
- 各工具调用与结果
- 最终整定结果
- 评估结果
- 历史经验参考摘要

### 2.2 经验中心

经验中心用于管理和查看系统已沉淀的整定经验。

支持的能力：

- 查看经验统计
- 按回路类型、模型类型、策略、是否通过进行筛选
- 相似经验检索
- 查看单条经验详情
- 清除全部历史经验
- 重建经验检索索引

## 3. 多智能体协作流程

系统包含 4 个主要智能体。

### 3.1 数据分析智能体

职责：

- 读取 CSV 或获取历史数据
- 进行列名标准化、时间解析和基础清洗
- 检测候选阶跃事件
- 生成候选辨识窗口

主要工具：

- `tool_fetch_history_data`
- `tool_load_data`

### 3.2 系统辨识智能体

职责：

- 基于候选窗口尝试多种过程模型
- 比较拟合质量与后续整定效果
- 选择最合适的过程模型

主要工具：

- `tool_fit_fopdt`

说明：

- 工具名为了兼容保留为 `tool_fit_fopdt`
- 实际内部已支持多模型辨识

### 3.3 PID 专家智能体

职责：

- 结合回路类型、模型类型和历史经验选择整定策略
- 生成基础 PID 参数
- 必要时优先尝试“历史修正版”参数

主要工具：

- `tool_search_experience`
- `tool_tune_pid`

### 3.4 评估智能体

职责：

- 对 PID 参数做闭环仿真评估
- 判断是否通过
- 在不通过时给出主因和回流目标
- 触发自动细调或切换其他候选模型窗口

主要工具：

- `tool_evaluate_pid`

## 4. 数据分析与候选窗口

系统不会直接用整段历史数据做辨识，而是先从历史数据中检测候选阶跃，再对每个候选事件生成辨识窗口。

当前处理步骤：

1. 标准化列名
2. 解析时间戳
3. 清洗缺失值
4. 检测候选阶跃
5. 为每个候选阶跃生成自适应窗口
6. 对多个候选窗口分别尝试辨识

这样做的目的是避免只靠一次固定窗口拍板，提高辨识稳定性。

## 5. 多模型辨识

### 5.1 当前支持的模型

- `FO`
- `FOPDT`
- `SOPDT`
- `IPDT`

### 5.2 原始模型参数与当前工作模型

系统辨识阶段会输出两层信息：

1. 原始模型参数  
   这是系统辨识智能体判断出的过程模型本体，例如：

- `FO`：`K, T`
- `FOPDT`：`K, T, L`
- `SOPDT`：`K, T1, T2, L`
- `IPDT`：积分增益 `K` 与 `L`

2. 当前整定阶段使用的工作模型  
   这是为了让整定与闭环试算稳定工作而采用的工作参数视图。

说明：

- 对 `FO` 和 `FOPDT`，工作模型与原始模型差异较小
- 对 `SOPDT`，当前整定阶段会保留主导与次级时间常数信息，但仍会构造一个便于整定计算的工作模型
- 对 `IPDT`，整定和评估已经按积分过程模型处理，不再简单退回普通一阶自衡模型

### 5.3 当前如何选模

系统辨识智能体会：

1. 根据 `loop_type` 设定候选模型优先级
2. 在候选窗口上分别拟合多个模型
3. 比较：
   - 拟合误差
   - `R²`
   - 模型置信度
   - 后续 PID 试算效果
4. 选择综合最优模型

最终结果中会包含：

- `modelType`
- `selectedModelParams`
- `modelSelectionReason`
- `tuningModel`

## 6. 按模型类型整定

当前整定层已经按模型类型分派，不再把所有模型统一当成 `FOPDT` 处理。

### 6.1 FO

对应整定方式：

- `FO` 保守 `Lambda / IMC`

特点：

- 适合快流量、快压力、弱滞后对象
- 不鼓励过于激进的策略

### 6.2 FOPDT

对应整定方式：

- `IMC`
- `LAMBDA`
- `ZN`
- `CHR`

这是当前最成熟的一条整定路径。

### 6.3 SOPDT

对应整定方式：

- `SOPDT` 主导/次级时间常数近似整定
- 优先 `Lambda / IMC`
- 受限条件下可尝试保守版 `ZN / CHR`

说明：

- 当前 `SOPDT` 已不再只是简单地把 `T1 + T2` 当成单一时间常数
- 整定阶段会保留 `T1/T2` 的主导与次级结构，并构造更合理的工作模型

### 6.4 IPDT

对应整定方式：

- 积分过程专用 `Lambda / IMC`

说明：

- `IPDT` 已有独立整定入口
- 闭环评估也已按积分过程真实模型仿真

## 7. 评分机制

系统不是根据整定方法名打分，而是根据闭环仿真效果打分。

### 7.1 第一层：性能分 `performance_score`

基于辨识模型和 PID 参数做闭环仿真，提取：

- 超调
- 调节时间
- 稳态误差
- 振荡次数
- 衰减比
- 是否稳定

当前权重大致为：

- 超调：25%
- 调节时间：20%
- 稳态误差：25%
- 振荡次数：15%
- 衰减比：15%

如果系统不稳定，还会再乘稳定性折减系数。

### 7.2 第二层：方法置信度 `method_confidence`

主要来自系统辨识阶段的模型置信度。

### 7.3 第三层：综合评分 `final_rating`

当前公式：

`final_rating = 0.7 × performance_score + 0.3 × (method_confidence × 10)`

当前通过阈值：

- `final_rating >= 7.0`

## 8. 自动回流优化

如果首次评估不通过，系统会自动分析主因并回流优化。

### 8.1 回流给 PID 专家

适用于：

- 超调过大
- 振荡明显
- 参数偏激进

动作：

- 自动收紧 `Kp / Ki`
- 重新评估

### 8.2 回流给系统辨识

适用于：

- 模型可信度不足
- 当前候选窗口不理想

动作：

- 切换其他候选窗口
- 重新辨识、整定和评估

## 9. 经验机制

### 9.1 经验沉淀

每次任务完成后，系统会自动生成结构化经验记录。

当前经验记录包括：

- 回路信息
- 数据来源与时间范围
- 原始模型类型与原始模型参数
- 初始 PID 与最终 PID
- 初始评估与最终评估
- 是否自动细调
- `refine_delta`
- `refine_gain`
- lessons 与标签

### 9.2 存储方式

当前采用双层存储：

- `jsonl`：事实源
- `sqlite`：检索索引

### 9.3 相似经验判断

当前相似度判断会考虑：

- `loop_type`
- `model_type`
- 原始模型参数是否接近
  - `FO`：`K / T`
  - `FOPDT`：`K / T / L`
  - `SOPDT`：`K / T1 / T2 / L`
  - `IPDT`：积分参数与 `L`
- 历史案例是否成功、评分是否高

### 9.4 经验复用方式

经验不会直接替代当前闭环试算，而是用于：

- 给出历史偏好策略
- 给出历史偏好模型
- 给出推荐的 `Kp / Ki / Kd` 收紧比例
- 优先生成“历史修正版 PID”候选

如果这些候选分数仍不够高，系统才继续展开更完整的 benchmark。

## 12. 使用建议

建议按下面顺序使用：

1. 先在“智能整定”页跑通一批真实任务
2. 再到“经验中心”观察经验沉淀情况
3. 对类似回路重复整定，逐步积累经验库
4. 结合相似经验检索，观察系统是否越来越偏向成功策略与成功修正模式

## 13. 当前仍在持续完善的部分

当前已经实现多模型辨识、多模型整定和模型类型化经验复用，但仍有一些持续优化空间：

- `SOPDT` 可继续向更原生的专用控制律推进
- 经验相似度可继续加入时间衰减与标签过滤
- 经验中心可继续增加趋势图和案例关联图
