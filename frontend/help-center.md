# PID 智能整定系统帮助文档

## 1. 系统定位

本系统用于工业控制回路的自动 PID 整定。

系统目标：

- 根据上传的 CSV 或历史数据，自动完成整定流程。
- 用多智能体协作方式完成数据分析、系统辨识、参数整定和效果评估。
- 把每次任务的成功经验沉淀下来，供后续类似回路参考。

系统当前支持：

- 任务过程展示
- 自动回流优化
- 经验中心
- 相似经验检索

## 2. 前端主要功能

### 2.1 智能整定页

智能整定页负责发起整定任务和查看执行过程。

支持的数据来源：

- 上传 CSV
- 获取历史数据

支持的回路类型：

- 流量
- 温度
- 压力
- 液位

任务执行时，页面会显示：

- 用户任务描述
- 多智能体执行过程
- 各工具调用与结果
- 最终整定结果
- 评估结果
- 经验参考摘要

### 2.2 经验中心

经验中心用于查看系统已沉淀的整定经验，便于前期测试和后续复用。

当前支持：

- 经验统计
- 经验列表
- 经验详情
- 相似经验检索
- 清除历史经验

## 3. 多智能体协作流程

当前系统包含 4 个主要智能体。

### 3.1 数据分析智能体

职责：

- 读取 CSV 或获取历史数据
- 做基础预处理
- 识别候选阶跃事件
- 选取辨识窗口

主要工具：

- `tool_fetch_history_data`
- `tool_load_data`

### 3.2 系统辨识智能体

职责：

- 基于候选窗口拟合 FOPDT 模型
- 给出 `K / T / L`
- 输出模型拟合质量与置信度

主要工具：

- `tool_fit_fopdt`

### 3.3 PID 专家智能体

职责：

- 在当前模型基础上选整定策略
- 计算 PID 参数
- 参考历史经验
- 必要时优先试“历史修正版”参数

主要工具：

- `tool_search_experience`
  - 前端过程里显示为经验检索
- `tool_tune_pid`

### 3.4 评估智能体

职责：

- 对 PID 参数做闭环仿真评估
- 判断是否通过
- 不通过时给出主因和下一步建议
- 必要时自动触发回流优化

主要工具：

- `tool_evaluate_pid`

## 4. 当前整定流程

标准流程如下：

1. 数据分析智能体完成数据准备
2. 系统辨识智能体完成 FOPDT 拟合
3. PID 专家智能体整定 PID 参数
4. 评估智能体评估闭环质量
5. 若不通过，系统自动回流优化
6. 若通过，沉淀本次经验

## 5. 数据分析与辨识机制

### 5.1 数据处理

系统会对输入数据执行：

- 列名标准化
- 时间戳解析
- 缺失值处理
- 基础清洗
- 候选阶跃识别

### 5.2 候选窗口

当前不会固定只用一个窗口，而是：

- 检出多个候选阶跃
- 为每个候选阶跃生成辨识窗口
- 对多个窗口分别做辨识
- 结合辨识质量与后续整定评估选择更优窗口

### 5.3 FOPDT 模型

系统辨识输出的一阶纯滞后模型参数为：

- `K`
  - 过程增益
- `T`
  - 时间常数
- `L`
  - 纯滞后时间

同时输出：

- 标准化 RMSE
- 原始 RMSE
- `R²`
- 模型置信度

## 6. PID 整定机制

### 6.1 策略集合

当前候选策略包括：

- `IMC`
- `LAMBDA`
- `ZN`
- `CHR`

### 6.2 当前策略选择逻辑

PID 专家不会只凭单条固定规则直接拍板，而是综合考虑：

- 回路类型
- 辨识模型质量
- 历史经验偏好
- 闭环试算结果

### 6.3 历史经验接入方式

经验检索会在 `tool_tune_pid` 前执行。

当前不仅会参考：

- 历史偏好策略

还会参考：

- 历史成功案例里 `Kp / Ki / Kd` 的收紧比例
- 历史成功案例的 refine pattern

如果历史经验显示：

- 类似回路通常在基础策略后还需要收紧 `Kp`
- 类似回路通常在基础策略后还需要收紧 `Ki`

那么系统会优先尝试：

- 基础策略参数
- 历史修正版参数

只有分数不够好时，才展开更完整的 benchmark。

## 7. 当前评分机制

系统的 PID 参数不是看方法名来评分，而是看闭环仿真效果。

### 7.1 第一层：闭环性能分 `performance_score`

基于辨识模型和 PID 参数进行阶跃仿真，提取：

- 超调
- 调节时间
- 稳态误差
- 振荡次数
- 衰减比
- 是否稳定

当前权重大致为：

- 超调：25%
- 调节时间：20%
- 稳态误差：25%
- 振荡次数：15%
- 衰减比：15%

若系统不稳定，会再乘稳定性折减系数。

### 7.2 第二层：方法置信度 `method_confidence`

当前主要来自系统辨识的模型置信度。

### 7.3 第三层：综合评分 `final_rating`

当前综合公式：

`final_rating = 0.7 × performance_score + 0.3 × (method_confidence × 10)`

当前通过阈值：

- `final_rating >= 7.0`

## 8. 自动回流优化机制

如果首次评估不通过，系统会先分析主因。

### 8.1 回流给 PID 专家

适用于：

- 超调过大
- 振荡明显
- 参数偏激进

动作：

- 自动收紧 `Kp / Ki`
- 重新评估

### 8.2 回流给系统辨识

适用于：

- 模型可信度不足
- 当前候选窗口不理想

动作：

- 比较其他候选辨识窗口
- 重新整定并评估

## 9. 经验机制

### 9.1 经验沉淀

每次任务完成后，系统会自动生成结构化经验记录。

当前经验内容包括：

- 回路信息
- 数据来源与时间范围
- 模型参数与拟合质量
- 初始 PID 参数
- 最终 PID 参数
- 初始评估与最终评估
- 是否自动细调
- 修正比例 `kp_scale / ki_scale / kd_scale`
- lessons 和标签

### 9.2 存储方式

当前经验使用双层存储：

- `jsonl`
  - 事实源
- `sqlite`
  - 检索索引

### 9.3 相似经验判断

当前相似度主要基于：

- 回路类型是否一致
- `K` 是否接近
- `T` 是否接近
- `L/T` 动态形态是否接近
- 历史经验本身是否成功、评分是否高

### 9.4 经验中心

经验中心用于：

- 查看经验统计
- 浏览经验列表
- 查看经验详情
- 按 `loop_type + K + T + L` 检索相似案例
- 清除全部历史经验（测试用）

## 12. 适用场景与注意事项

### 12.1 适用场景

- 回路历史数据较完整
- 存在明显阶跃或可辨识动态
- 需要快速形成可解释的 PID 初值

### 12.2 当前注意事项

- 辨识质量仍然决定整定质量上限
- 历史经验是增强项，不替代当前闭环评估
- 类似回路经验越多，后续参考价值越高
- “清除历史经验”功能主要用于前期测试，生产环境不建议频繁使用

## 13. 建议的使用方式

建议按下面顺序使用：

1. 先在“智能整定”页跑通一批真实任务
2. 到“经验中心”观察经验沉淀情况
3. 对类似回路反复整定，逐步积累经验库
4. 结合经验中心和相似经验检索，观察系统是否越来越倾向成功策略与成功修正模式

## 14. 后续扩展建议

后续建议优先增加：

- 经验时间衰减
- 标签过滤
- 更强的相似经验聚类
